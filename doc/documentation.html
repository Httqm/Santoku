What is Santoku ?

Santoku is a Python script designed to generate configuration files for Shinken and Nagios from data provided in a CSV file. The CSV file is expected to have a specific format. The generated files are :

  • hosts.cfg, including host groups
  • services.cfg
  • commands.cfg
  • and a very basic tabular-presented Nagvis map showing ALL hosts listed in the CSV file (0% done)

At a previous position, I wrote a PERL script doing all this (except the commands.cfg part). This script still works very well, but because of the numerous plugins / environments / usages it handles, it's now a 60KBytes cluttered script and debugging/updating it has become a nightmare. So I can't even share it with the community.
So I started developping Santoku :

  • because I wanted to build a modular tool that others could benefit from, without being developers themselves
  • to improve my Python skills
  • ... and because coding is fun ;-)
The concept

Basically speaking, just consider a few things :

  • a CSV file you can generate with a spreadsheet programm such as (Open|Libre)Office Calc or Microsoft Excel
  • the Santoku script
  • a configuration file for every plugin you're using

Why "Santoku" ?

Well, ... there was Batman and Robin. Now, there is Shinken and Santoku :-)
Actually, since Santoku is designed to be used together with Shinken, I wanted to name it as a Japanese cutting device. However, since Santoku is a very humble piece of software, I could not name it as a deadly weapon.
A santoku is a japanese kitchen knife.

Requirements

Where does Santoku work ?

Santoku is developed on a Debian Squeeze workstation, with Python 2.6, and using CSV files made with Calc from OpenOffice.org 3.2.1 . I expect some minor fixes to support CSV files made with Microsoft Excel, and to handle files made in a Microsof Windows environment (line end encoding...). To be done.

Usage

How to get Santoku ?

git clone https://github.com/Httqm/Santoku.git

Installation

Simply move the Santoku folder you got with git clone to the desired location. You're done !

Running Santoku

Santoku has no command-line flag since every option it requires it stored in its configuration file modules/config.py.
To run Santoku, just type ./santoku.py

Known bugs and limitations

Since Santoku is in its early version, few bugs and limitations are known so far. Here are some important notices :

  • In the commands.cfg file, Santoku writes the commands related to the plugins execution. In a Shinken fresh install, the commands.cfg file not only stores the plugin commands, but also the notification commands. So overwriting the default commands.cfg will erase the notification commands. To fix this :
    1. create a commands_notification.cfg file in the folder containing the commands.cfg file on the host running Shinken
    2. save the notification commands "notify-host-by-email" and "notify-service-by-email" in commands_notification.cfg
    3. register commands_notification.cfg in nagios.cfg by declaring it like commands.cfg : cfg_file = commands_notification.cfg
  • Santoku is designed so that, upon error, it exits cleanly with a detailed error message. If it happens to die on a Python error : this is a BUG ! Please report an issue.
